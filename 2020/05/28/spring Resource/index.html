spring Resource

May 28 2020

Resource是对资源的抽象，它的每一个实现类都代表了一种资源的访问策略。如FileSystemResource、ByteArrayResource等。

Resource作为所有资源统一抽象，它定义了一些通用的方法，由子类 AbstractResource 提供统一的默认实现。

其中Resource根据资源的不同类型有不同的实现

1.AbstractResource是Resource 接口的默认抽象实现。它实现了 Resource 接口的大部分的公共实现，是Resource接口中的重点

2.FileSystemResource是对java.io.File类型的资源的封装

3.ByteArrayResource是对字节数组提供的数据的封装。如果通过 InputStream 形式访问该类型的资源，该实现会根据字节数组的数据构造一个相应的 ByteArrayInputStream。

4.UrlResource是对java.net.URL类型资源的封装。内部委派 URL 进行具体的资源操作。

5.ClassPathResource是class path 类型资源的实现。使用给定的 ClassLoader 或者给定的 Class 来加载资源。

6.InputStreamResource是将给定的 InputStream 作为一种资源的 Resource 的实现类。
